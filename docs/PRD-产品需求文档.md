# 「世间万物」PRD 产品需求文档

版本：v2.0  
日期：2026-05-27

## 1. 产品定位

「世间万物」是一个中文泛知识学习及书籍销售网站。它以 10 大篇章组织 100 个知识领域，为每个领域提供入门、进阶、专家三个阶段的学习路径，并通过推荐书单与微信视频号/微信小店二维码实现销售转化。

产品核心价值：

- 帮助用户在广阔知识领域中快速找到学习入口。
- 以阶段化内容降低通识学习门槛。
- 用结构化知识模块、练习和笔记形成学习闭环。
- 通过推荐书单连接学习内容与购买场景。

## 2. 产品愿景

网站名称：世间万物  
Slogan：探索知识的边界，从入门到精通  
目标用户：泛知识学习者、终身学习者、图书购买用户、知识内容运营者  
核心承诺：每个领域都有清晰路径，每个阶段都有可学习内容，每份内容都能指向下一步阅读和实践。

## 3. 核心架构

| 层级 | 数量 | 说明 |
|---|---:|---|
| 篇章 | 10 | 主题篇章 |
| 领域 | 100 | 每篇 10 个领域 |
| 阶段 | 3 | 入门、进阶、专家 |
| 内容 JSON | 300 | 100 领域 × 3 阶段 |
| 推荐书目 | 每阶段 3-6 条 | 可购买或待补充 |
| 搜索索引 | 自动生成 | 来源于 300 份内容 |

## 4. 用户路径

### 4.1 新用户路径

1. 进入首页，理解平台定位。
2. 点击“开始学习”进入探索页。
3. 通过篇章筛选或搜索找到领域。
4. 进入领域详情页，查看学习路径。
5. 选择入门阶段开始学习。
6. 阅读模块，点击“我已理解”记录进度。
7. 完成练习，查看推荐书目。
8. 扫描微信小店二维码购买相关书籍。

### 4.2 老用户路径

1. 进入个人面板。
2. 查看已学习阶段和成就徽章。
3. 返回上次学习领域。
4. 继续记录笔记和完成阶段。

### 4.3 购买路径

1. 用户在领域详情页或阶段学习页看到推荐书单。
2. 点击购买按钮。
3. 若为链接购买，打开外部购买链接。
4. 若为二维码购买，弹出二维码 Dialog。
5. 若暂未补充，显示“购买链接待补充”。

## 5. 页面需求

### 5.1 首页

路由：`#/`

必须包含：

- 品牌名称。
- Slogan。
- Hero 区域。
- CTA：开始学习。
- 平台统计：100 知识领域、300 学习内容、10 大篇章。
- 十篇章概览。
- 精选领域。

验收：

- CTA 跳转 `#/explore`。
- 统计数字来自实际数据或与实际数据保持一致。

### 5.2 探索页

路由：`#/explore`

必须包含：

- 100 个领域卡片。
- 篇章筛选。
- 领域搜索。
- 收藏功能。

验收：

- 所有领域卡片的链接为 `#/domain/{domainId}`。
- 任意篇章筛选后只显示该篇章领域。
- 搜索领域标题可返回结果。

### 5.3 领域详情页

路由：`#/domain/:id`

必须包含：

- 领域标题、描述、篇章。
- 难度或学习建议。
- 三阶段路径。
- 当前领域推荐书目。
- 微信小店二维码入口。
- 相关领域推荐。

验收：

- 页面标题与 `domains.json` 完全一致。
- 三阶段入口分别指向：
  - `#/domain/{id}/stage/beginner`
  - `#/domain/{id}/stage/intermediate`
  - `#/domain/{id}/stage/advanced`

### 5.4 阶段学习页

路由：`#/domain/:id/stage/:stageKey`

这是平台核心页面。

必须包含：

- 面包屑导航。
- 阶段标题。
- 阶段导言。
- 学习内容清单。
- 5-8 个可展开知识模块。
- 每个知识模块底部有“我已理解”。
- 3-5 道练习，支持查看/收起答案。
- 推荐书目。
- 微信小店二维码。
- 小店橱窗入口。
- 学习笔记。
- 上一阶段/下一阶段导航。

验收：

- 页面标题、URL、JSON 内容三者一致。
- 不允许加载到其他领域内容。
- 不允许显示“暂无该阶段的学习内容”。

### 5.5 个人面板

路由：`#/dashboard`

必须包含：

- 学习统计概览。
- 环形进度条。
- 五级成就徽章。
- 学习足迹。
- 推荐领域。
- 快捷入口。

徽章名称：

- 好奇小白
- 探索达人
- 精进修习者
- 领域大师
- 智慧长者

### 5.6 成长路线

路由：`#/roadmap`

必须包含：

- 五阶成长全景地图。
- 逐阶详解。
- 十篇章之旅导航。

### 5.7 搜索页

路由：`#/search`

必须包含：

- 搜索输入框。
- 推荐关键词。
- 搜索结果。
- 关键词高亮。
- 点击进入对应阶段页。

搜索范围：

- 篇章名称。
- 领域名称。
- 阶段名称。
- 知识点标题。
- 知识点摘要。
- 知识点正文摘录。
- 关键词。

### 5.8 管理后台

路由：`#/admin`

必须包含：

- 密码保护。
- 数据统计总览。
- 领域/阶段状态概览。
- 快速跳转。

说明：GitHub Pages 是静态托管，后台不能直接在线写入 GitHub 文件。后台定位是查看、辅助维护与运营检查。

## 6. 非功能需求

- GitHub Pages 可访问。
- 使用 HashRouter。
- 路由刷新不 404。
- 所有静态资源使用 `/wanwuzhi/` 前缀。
- 数据使用 JSON 静态加载。
- 学习进度与笔记使用 `localStorage`。
- 移动端和桌面端均可用。
- 发布前必须通过自动验收。

## 7. 100% 完成定义

满足以下条件才可称为 100% 需求版：

- 100 个领域全部可访问。
- 300 个阶段页面全部可访问。
- 300 个阶段页面内容全部匹配对应领域。
- 每个阶段都有真实可学习内容。
- 搜索功能可用。
- 书籍推荐可按领域和阶段匹配。
- 微信小店二维码可展示。
- 顶部导航无错误路由。
- 发布流程有校验和回滚方案。
