# 「世间万物」内容生产规范

版本：v2.0  
日期：2026-05-27

## 1. 内容目标

每个阶段页面都必须是用户真正可以学习的材料，而不是占位内容、通用模板或关键词拼接。

内容目标：

- 帮助用户理解领域核心问题。
- 建立从入门到专家的递进路径。
- 提供可展开的知识模块。
- 提供练习、任务和推荐阅读。
- 能自然引导到书籍购买。

## 2. 内容层级

```plain
篇章
  领域
    入门阶段
    进阶阶段
    专家阶段
```

每个阶段是一份独立 JSON，每份 JSON 是一个完整学习单元。

## 3. 阶段定位

### 3.1 入门阶段

适合初学者。

内容重点：

- 基本概念。
- 核心问题。
- 关键人物或理论。
- 常见误区。
- 入门实践。

写作要求：

- 避免过度学术化。
- 多用例子。
- 帮助用户建立兴趣和地图感。

### 3.2 进阶阶段

适合已有基础的学习者。

内容重点：

- 理论模型。
- 方法框架。
- 案例分析。
- 实践迁移。
- 多观点比较。

写作要求：

- 强调结构和方法。
- 引入真实案例。
- 训练用户分析能力。

### 3.3 专家阶段

适合深度学习者。

内容重点：

- 前沿争议。
- 跨学科连接。
- 高阶问题。
- 综合应用。
- 创造性输出。

写作要求：

- 有深度和边界意识。
- 能呈现不同立场。
- 鼓励用户形成自己的判断。

## 4. 每份内容 JSON 的最低要求

- 阶段导言：500-1000 字。
- 知识模块：5-8 个。
- 练习：3-5 道。
- 阶段任务：2-4 个。
- 推荐资源：3-6 条。

## 5. 阶段导言写法

导言必须回答：

- 这个阶段学习什么？
- 为什么这个阶段重要？
- 用户学完会获得什么能力？
- 和上一阶段/下一阶段是什么关系？
- 建议如何学习？

禁止：

- 空泛鼓励。
- 重复同一段话。
- 只替换领域名。
- 出现英文占位词或未翻译内容。

## 6. 知识模块写法

每个模块必须包含：

- 标题。
- 摘要。
- 正文。
- 案例。
- 核心要点。
- 延伸思考。

正文建议结构：

1. 概念定义。
2. 背景或来源。
3. 关键机制。
4. 实际例子。
5. 常见误区。
6. 学习建议。

## 7. 练习写法

练习必须服务于当前阶段能力。

题型：

- 理解：检验概念掌握。
- 应用：把知识用于具体场景。
- 分析：比较、拆解、判断。
- 反思：连接自身经验。
- 创作：输出作品、方案或观点。

每道题必须有参考答案。参考答案不是唯一标准答案，而是帮助用户理解评分方向。

## 8. 推荐资源写法

推荐资源应优先选择：

- 经典书籍。
- 权威入门教材。
- 领域代表作品。
- 可靠课程或公开资源。
- 可用于实践的工具。

每条资源必须说明：

- 为什么推荐。
- 适合哪个阶段。
- 读者能获得什么。

## 9. 去模板化要求

以下内容不得发布：

- “在某某领域的学习中，理论和实践结合至关重要”反复出现。
- 多个领域只替换领域名，正文结构完全相同。
- 案例与领域无关。
- 练习题只要求“写一篇 800 字报告”，没有领域特异性。
- 资源标题为“某某经典指南”但没有真实书名或推荐理由。

检查方法：

- 抽查同一篇章下 10 个领域，模块标题不得高度重复。
- 抽查同一领域三阶段，内容必须递进。
- 抽查任意 20 个阶段，不得出现明显批量生成痕迹。

## 10. 内容重写优先级

### P0：修正错配

确保 `content/{domainId}` 与 `domains.json` 中同 ID 的领域完全一致。

### P1：补齐缺字段

修复缺少 `domainId`、`domainTitle`、`stageKey`、`chapterId` 的内容。

### P2：重写模板化内容

优先重写 `61-100` 号领域。

### P3：提升深度

为每个阶段增加更好的案例、练习和推荐资源。

## 11. 内容验收评分

每份阶段内容按 5 分制评分：

- 5 分：内容完整、专属、可学习，有案例和练习。
- 4 分：内容完整，少量段落可继续优化。
- 3 分：结构完整，但深度一般。
- 2 分：模板化明显，仅可临时占位。
- 1 分：错配、空泛或不可学习。

正式上线要求：

- P0 发布：不得低于 3 分。
- 100% 需求版：不得低于 4 分。
