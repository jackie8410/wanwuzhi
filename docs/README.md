# 「世间万物」项目文档中心

本目录用于管理「世间万物」知识探索平台达到 100% 项目需求所需的全部核心文档。

## 文档阅读顺序

1. [世间万物-100%需求重建总文档.md](./世间万物-100%需求重建总文档.md)  
   项目总纲，定义重建目标、问题基线、修正顺序与交付里程碑。

2. [PRD-产品需求文档.md](./PRD-产品需求文档.md)  
   定义产品定位、页面功能、用户路径、学习与销售闭环。

3. [DATA-数据模型与Schema.md](./DATA-数据模型与Schema.md)  
   定义章节、领域、阶段、内容、书籍、搜索索引的数据契约。

4. [TECH-技术架构文档.md](./TECH-技术架构文档.md)  
   定义前端架构、数据加载、路由、状态存储、构建与脚本体系。

5. [CONTENT-内容生产规范.md](./CONTENT-内容生产规范.md)  
   定义 300 份学习内容的写作标准、质量门槛与去模板化要求。

6. [BOOKS-书籍销售与二维码规范.md](./BOOKS-书籍销售与二维码规范.md)  
   定义推荐书目、微信小店二维码、购买按钮状态与销售数据维护规则。

7. [QA-全覆盖验收清单.md](./QA-全覆盖验收清单.md)  
   定义发布前必须通过的数据、页面、搜索、二维码和 GitHub Pages 验收项。

8. [DEPLOY-GitHub部署与更新流程.md](./DEPLOY-GitHub部署与更新流程.md)  
   定义 GitHub Pages 静态部署、更新、回滚和发布检查方法。

9. [IMPLEMENTATION-100%落地实施方法.md](./IMPLEMENTATION-100%落地实施方法.md)  
   定义从当前 V1 框架版到 100% 需求版的具体实施路线。

## 核心原则

- 先统一数据主键，再修前端页面。
- 先解决内容错配，再优化 UI。
- 先自动校验，再发布 GitHub Pages。
- `domainId + stageKey` 是学习内容和书籍推荐的核心绑定方式。
- 任何正式发布都必须通过 QA 文档定义的 P0 阻断项检查。
