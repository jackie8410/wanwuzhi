# 「世间万物」知识探索平台 100%需求重建总文档

版本：v2.0 重建版  
日期：2026-05-27  
适用范围：`wanwuzhi` GitHub Pages 静态网站、内容数据、书籍销售数据、发布验收流程

## 1. 重建目标

「世间万物」是一个中文泛知识学习与书籍销售平台，目标是以 10 大篇章组织 100 个知识领域，为每个领域提供入门、进阶、专家三个阶段的系统学习内容，并通过微信视频号/微信小店二维码完成书籍或学习资源销售转化。

本次重建不是局部修补，而是把当前 V1 框架版升级为可持续维护、可自动验收、可稳定部署的正式内容平台。

最终验收目标：

- 10 大篇章，每篇 10 个领域，合计 100 个领域。
- 每个领域 3 个阶段，合计 300 份学习内容 JSON。
- 每份学习内容都能在 GitHub Pages 上正确加载、正确匹配领域与阶段。
- 每个阶段页面包含阶段导言、学习模块、练习巩固、推荐书单、微信小店二维码、学习笔记和阶段导航。
- 书籍销售数据可按 `domainId + stageKey` 准确匹配，不再依赖不稳定的中文名称匹配。
- 搜索索引从最终内容自动生成，结果可点击进入对应阶段页面。
- 发布前必须通过自动验收脚本，禁止空内容、错内容、错路由、缺二维码、缺字段直接上线。

## 2. 当前问题基线

根据本轮审计，当前项目不是“没有框架”，而是“框架与数据没有统一主键”。主要问题如下。

### 2.1 文件数量达标，但内容不可直接视为达标

- `content/1` 到 `content/100` 存在。
- 每个目录有 `beginner.json`、`intermediate.json`、`advanced.json`。
- 合计 300 个阶段 JSON，数量达标。
- `search-index.json` 有 1632 条索引，数量与全部知识点相等。
- `data/books.json` 有 496 条书籍数据。
- `qr/store-qr.png` 存在。

但数量达标不等于内容达标。当前最严重的问题是领域显示数据、内容 JSON、搜索索引、书籍数据之间没有同一个稳定 ID。

### 2.2 领域与内容大面积错配

线上探索页显示的领域标题与 `content/{id}/{stage}.json` 的内容标题大量不一致。典型问题：

- URL：`#/domain/100/stage/intermediate`
- 页面标题：`宇宙观与人类未来 · 进阶阶段`
- 实际导言与模块内容：`对生命说谢谢的能力`

这说明当前系统不是单纯加载失败，而是加载到了错误主题的内容。

### 2.3 JSON schema 不统一

当前 300 个内容 JSON 至少存在多套结构。严重问题包括：

- 部分文件缺少 `domainId`。
- 部分文件缺少 `domainTitle`。
- 部分文件缺少 `stageKey` 或 `stage`。
- 部分资源使用 `desc`，部分资源使用 `description`。
- 部分标题把阶段写进 `title`，部分标题不写。
- 部分阶段知识点少于 5 个，不满足需求。

### 2.4 内容质量不稳定

当前内容大致分为四类：

- `1-20`：部分内容相对细，但仍存在标题和结构规则不统一。
- `21-40`：内容主体存在，但关键元数据缺失。
- `41-60`：结构较完整，但资源字段不统一。
- `61-100`：模板化严重，领域专属性不足。

因此后续不能只做字段修补，还必须做内容质量分级与重写。

### 2.5 书籍销售闭环未完成

`books.json` 虽有 496 条，但问题明显：

- 大量 `ExplorationDomain` 为空。
- 全部 `purchaseUrl` 为空。
- 全部 `purchaseQrCode` 为空。
- 大量 `author` 为空。
- 大量 `description` 为空。

当前只能说“全局微信小店二维码可展示”，不能说“每本书可准确扫码购买”。

### 2.6 前端路由与搜索仍有缺陷

- 顶部“学习路线”链接到 `#/learning-path`，实际路由是 `#/roadmap`。
- 顶部“我的学习”链接到 `#/my-learning`，实际路由是 `#/dashboard`。
- 搜索页 UI 能显示，但输入关键词后未稳定返回结果。
- 即使搜索功能修复，当前索引也会继承错配内容，因此必须在内容修复后重新生成索引。

## 4. 重建顺序

本项目必须按以下顺序修复。跳过前置步骤会导致后续继续错配。

### 阶段 1：建立唯一权威数据源

目标：所有页面、内容 JSON、书籍数据、搜索索引都必须使用同一套 `domainId` 与 `stageKey`。

必须产出：

- `data/chapters.json`
- `data/domains.json`
- `data/stages.json`

`domains.json` 是 100 个领域的唯一权威清单。前端不得再在多个文件中维护领域标题。

验收标准：

- `domains.json` 恰好 100 条。
- 每个 `domainId` 在 1-100 范围内且唯一。
- 每个 `chapterId` 恰好绑定 10 个领域。
- 每个领域有稳定字段：`domainId`、`chapterId`、`domainTitle`、`subtitle`、`description`、`difficulty`、`keywords`。

### 阶段 2：统一 300 份学习内容 JSON schema

目标：每份内容 JSON 都可以独立校验和独立渲染。

路径规范：

```plain
content/{domainId}/{stageKey}.json
```

示例：

```plain
content/100/intermediate.json
```

统一 schema：

```json
{
  "schemaVersion": "2.0",
  "domainId": 100,
  "chapterId": 10,
  "domainTitle": "宇宙观与人类未来",
  "chapterTitle": "宇宙规律与哲学思辨",
  "stageKey": "intermediate",
  "stageLabel": "进阶阶段",
  "title": "宇宙观与人类未来 · 进阶阶段",
  "intro": "500-1000字阶段导言",
  "knowledgePoints": [],
  "exercises": [],
  "tasks": [],
  "resources": []
}
```

字段规则：

- `domainId` 必须与路径目录一致。
- `stageKey` 必须与文件名一致。
- `domainTitle` 必须与 `data/domains.json` 中同 ID 标题完全一致。
- `chapterId` 必须与 `data/domains.json` 中同 ID 的 `chapterId` 一致。
- `title` 只能由 `{domainTitle} · {stageLabel}` 生成。
- 不允许 `domainName`、`chapterName`、`stage` 等旧字段作为主字段继续扩散。

### 阶段 3：重建内容质量标准

每份阶段内容必须包含：

- 阶段导言：500-1000 字。
- 学习内容清单：5-8 个知识模块。
- 每个知识模块：
  - `id`
  - `title`
  - `summary`
  - `content`
  - `examples`
  - `keyTakeaways`
  - `furtherThinking`
- 练习巩固：3-5 道题。
- 阶段任务：2-4 个。
- 推荐资源：3-6 条。

知识模块 schema：

```json
{
  "id": "kp-100-intermediate-01",
  "title": "宇宙尺度下的人类位置",
  "summary": "一句话说明本模块解决什么问题。",
  "content": "系统学习正文。",
  "examples": [
    "案例或类比 1",
    "案例或类比 2"
  ],
  "keyTakeaways": [
    "要点 1",
    "要点 2",
    "要点 3"
  ],
  "furtherThinking": [
    "思考题 1",
    "思考题 2"
  ]
}
```

练习 schema：

```json
{
  "id": "ex-100-intermediate-01",
  "type": "理解",
  "question": "题目文本",
  "answer": "参考答案",
  "hint": "可选提示"
}
```

允许的练习类型：

- 理解
- 应用
- 分析
- 反思
- 创作

### 阶段 4：重建书籍与销售数据模型

目标：推荐书籍从中文名称匹配改为 ID 精确绑定。

新文件：

```plain
data/books.json
```

统一 schema：

```json
{
  "bookId": "B100-I-001",
  "domainId": 100,
  "stageKey": "intermediate",
  "title": "书名",
  "author": "作者",
  "publisher": "出版社，可选",
  "description": "推荐理由",
  "resourceType": "book",
  "purchaseType": "store_qrcode",
  "purchaseUrl": "",
  "purchaseQrCode": "/wanwuzhi/qr/store-qr.png",
  "priority": 1
}
```

`purchaseType` 可选值：

- `url`：跳转外部购买链接。
- `book_qrcode`：展示单本书二维码。
- `store_qrcode`：展示统一微信小店二维码。
- `pending`：购买链接待补充。

阶段页书籍匹配规则：

```plain
domainId == 当前领域 ID
stageKey == 当前阶段 key
```

领域详情页书籍匹配规则：

```plain
domainId == 当前领域 ID
```

不得再使用 `ExplorationDomain` 中文字符串作为唯一匹配依据。

### 阶段 5：重建搜索索引

搜索索引必须由校验通过后的 300 份内容 JSON 自动生成。

索引路径：

```plain
content/search-index.json
```

索引条目 schema：

```json
{
  "domainId": 100,
  "domainTitle": "宇宙观与人类未来",
  "chapterId": 10,
  "chapterTitle": "宇宙规律与哲学思辨",
  "stageKey": "intermediate",
  "stageLabel": "进阶阶段",
  "kpId": "kp-100-intermediate-01",
  "kpTitle": "宇宙尺度下的人类位置",
  "kpSummary": "摘要",
  "content": "用于搜索的正文摘录",
  "keywords": ["宇宙观", "人类未来", "科学哲学"],
  "url": "/domain/100/stage/intermediate"
}
```

搜索验收：

- 输入领域名能搜到该领域。
- 输入篇章名能搜到相关领域。
- 输入知识点标题能定位到对应阶段。
- 搜索结果点击后进入正确 hash 路由。
- 搜索结果不得出现空 `domainTitle`。

### 阶段 6：修复前端路由与页面引用

必须统一路由：

| 页面 | 正确路由 |
|---|---|
| 首页 | `/` |
| 探索页 | `/explore` |
| 领域详情页 | `/domain/:id` |
| 阶段学习页 | `/domain/:id/stage/:stageKey` |
| 个人面板 | `/dashboard` |
| 成长路线 | `/roadmap` |
| 搜索页 | `/search` |
| 管理后台 | `/admin` |

顶部导航必须指向：

- 首页：`#/`
- 探索领域：`#/explore`
- 学习路线：`#/roadmap`
- 我的学习：`#/dashboard`

不得继续使用：

- `#/learning-path`
- `#/my-learning`

### 阶段 7：建立自动验收脚本

发布前必须运行以下校验。

#### 数据校验

检查项：

- 100 个领域。
- 10 个篇章。
- 每篇章 10 个领域。
- 300 个内容 JSON。
- 每个内容 JSON 字段齐全。
- `domainId` 与路径一致。
- `stageKey` 与文件名一致。
- `domainTitle` 与 `domains.json` 一致。
- 每阶段 5-8 个知识点。
- 每阶段 3-5 道练习。
- 练习必须有 `question` 和 `answer`。
- 资源字段必须统一为 `description`。
- 不允许旧字段 `desc` 作为唯一描述字段。

#### 质量校验

检查项：

- 不允许 `暂无`、`待补充`、`TODO` 出现在正式内容中。
- 不允许推荐资源 URL 全部为 `#`。
- 不允许 `domainTitle` 为空。
- 不允许 `bookId` 为空。
- 不允许书籍 `domainId` 指向不存在的领域。
- 不允许书籍 `stageKey` 使用非法值。
- 不允许页面标题与 JSON 标题不一致。

#### 线上路由校验

程序化访问：

```plain

/explore
/domain/1
/domain/100
/domain/1/stage/beginner
...
/domain/100/stage/advanced
/dashboard
/roadmap
/search
/admin
```

300 个阶段页必须验证：

- 页面不空白。
- 不显示“暂无该阶段的学习内容”。
- H1 与 `domainTitle + stageLabel` 一致。
- 有阶段导言。
- 有学习内容清单。
- 有练习巩固。
- 有推荐书目。
- 有微信小店入口。
- 有学习笔记。
- 上一阶段/下一阶段按钮正确。

## 6. 数据主键原则

本项目所有数据都必须遵守以下规则：

- `domainId` 是领域唯一主键。
- `chapterId` 是篇章唯一主键。
- `stageKey` 是阶段唯一主键。
- `bookId` 是书籍唯一主键。
- `kpId` 是知识点唯一主键。
- `exerciseId` 是练习唯一主键。

禁止：

- 用中文领域名作为唯一关联键。
- 一个领域在不同文件中使用不同标题。
- 一个阶段同时使用 `advanced`、`expert`、`master` 等多个 key。
- 手动维护搜索索引。
- 发布未通过校验的数据。

## 7. 内容重写优先级

### P0：先修错配

优先修复全部 `domainId` 与内容主题错配问题。错配未修复前，不应继续优化 UI。

### P1：补齐元数据

修复 `21-40` 号领域缺失元数据问题。

### P2：重写模板内容
